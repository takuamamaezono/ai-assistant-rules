# Claude とのやり取りルール

## 🔐 セキュリティ最優先原則

**すべての作業においてセキュリティを第一優先とします**

### セキュリティ優先事項
1. **機密情報の保護**
   - APIキー、パスワード、トークンなどの機密情報を絶対にログに出力しない
   - 秘密情報をコードに直接記載しない（環境変数やシークレット管理を使用）
   - ユーザーの個人情報を適切に保護する

2. **最新セキュリティ情報の確認**
   - 新しいライブラリ導入前に既知の脆弱性をチェック
   - セキュリティホールが発見された場合は即座に対応
   - 2025年時点の最新セキュリティベストプラクティスを適用

3. **防御的プログラミング**
   - 入力値の検証を必ず実施
   - SQLインジェクション、XSSなど一般的な攻撃への対策
   - 権限管理とアクセス制御の適切な実装

4. **セキュリティ情報の更新**
   - セキュリティ関連の知識は常に最新情報で更新
   - CVE データベース、OWASP ガイドラインなどを参照
   - AI時代特有のセキュリティリスクも考慮

### 作業中断基準
以下の場合は作業を中断し、セキュリティリスクを報告：
- 機密情報が露出する可能性がある場合
- 未知のセキュリティリスクが発見された場合
- 安全でないコードパターンが検出された場合
